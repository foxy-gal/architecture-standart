## Крупные задачи для будущего планирования (по системам)

### Депозитная платформа MVP
1. Спроектировать контракт **Rates API**: поля, фильтры, дата актуальности, версия, справочники (валюта/срок), коды продуктов.
2. Реализовать **аутентификацию/авторизацию** для потребителя «Кол-центр банка» (IAM, роли, политики).
3. Добавить **аудит и логирование** доступа к ставкам (кто/когда/что).
4. Реализовать **Rate Exporter**:
   - генерация снапшота ставок (CSV или JSON) + `manifest.json`,
   - версионирование формата,
   - расчёт `hash` / контроль целостности,
   - ретраи / идемпотентность,
   - хранение последнего успешного снапшота.
5. Интегрироваться с каналом доставки (**MFT/защищённая почта**): отправка, статусы, алёрты.
6. **Наблюдаемость**: метрики по API (latency/error rate), метрики по экспорту (успех/ошибка/время), алёрты, дашборды.
7. **Нагрузочное тестирование** (пик кампании) и кэш-стратегия.

### Система кол-центра банка
1. Доработка UI: раздел/виджет **«Актуальные ставки»** + удобный поиск (валюта/срок/сумма).
2. Доработка бэкенда: интеграция с Rates API, **кэш** и деградация (показ последнего значения при сбое).
3. Настроить **сервисную авторизацию** (учётка клиента, секреты/сертификаты, ротация).
4. **Логи/аудит** консультаций (минимально — факт запроса ставок и версия).

### Шлюз файлового обмена / защищённая почта (инфраструктура банка)
1. Выбрать и согласовать канал доставки (MFT или защищённая почта) под требования ИБ и партнёра.
2. Настроить шифрование (PGP/S-MIME при почте) и обмен ключами/сертификатами.
3. Настроить мониторинг доставки, ретраи, журналирование.

### Партнёрский кол-центр (внешняя система)
1. Согласовать формат файла, справочники, частоту доставки, SLA по импорту.
2. Реализовать импорт и валидацию файла (версия схемы, дата актуальности, контрольная сумма).
3. Процедуры на случай пропуска файла/ошибки импорта (эскалация, ручной импорт).

### Сквозные организационные задачи
1. Зафиксировать «единый источник текущих ставок» и ответственность за актуальность (владельцы данных/процесса).
2. Подготовить тестовые файлы и контрактные тесты (включая «сломанные» кейсы).
3. Runbook для поддержки: что делать при сбое экспорта/доставки/импорта, и кто on-call.
